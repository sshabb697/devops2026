# 08 - Backlogs

## Overview

A **backlog** is a prioritized list of work. In Azure Boards, backlogs exist at multiple levels (Epics, Features, Stories) and are where you **add**, **order**, **estimate**, and **plan** work before it enters a sprint.

## Backlog Levels

The Agile process provides three backlog levels:

| Level | Contains | Audience |
| ----- | -------- | -------- |
| **Epics** | Epics | Leadership / roadmap |
| **Features** | Features | Product managers |
| **Backlog (Stories)** | User Stories (+ Bugs) | The dev team |

Switch levels using the selector at the top of the backlog view.

## What You Do on a Backlog

- **Add** new items quickly (one per row).
- **Prioritize** by dragging to reorder (sets the **stack rank**).
- **Estimate** with story points.
- **Assign** area/iteration, owner, tags.
- **Decompose** — expand a parent to see/add children.
- **Map** parents (e.g., assign Stories to Features in the mapping pane).
- **Forecast** — project which items fit upcoming sprints based on velocity.

## Prioritization (Stack Ranking)

The order of the backlog *is* the priority. Drag items up/down; the top is most important. This **stack rank** drives what the team pulls into sprints next.

## Backlog vs Board

| | Backlog | Board (Kanban) |
| --- | ------- | -------------- |
| View | Ordered list | Columns of cards |
| Best for | Prioritizing, estimating, planning | Visualizing flow, updating status |
| Order matters | Yes (priority) | Within columns |

They show the **same items** in different views.

## Backlog Tools (right-side panes)

- **View options:** toggle **Parents**, **Forecasting**, **In Progress items**, **Completed child items**.
- **Mapping:** drag Stories onto Features to set parentage.
- **Planning:** drag items into upcoming **sprints**.

## Forecasting

With a velocity value entered, **Forecasting** draws lines showing how far down the backlog the team is likely to complete in each future sprint — a quick capacity sanity check.

## Backlog Refinement (Grooming)

A recurring activity (not a formal Scrum event) where the team:
- Clarifies and splits items.
- Adds/updates acceptance criteria.
- Estimates new items.
- Re-prioritizes.

A well-refined backlog makes sprint planning fast.

## Best Practices

- Keep the **top of the backlog** well-refined and estimated ("ready").
- Order strictly by priority/value.
- Don't over-detail items far down the backlog (they'll change).
- Use **tags** and **area paths** to filter large backlogs.
- Refine regularly so sprint planning is smooth.

## Summary

- A backlog is a prioritized list of work, available at Epic/Feature/Story levels.
- The list **order is the priority** (stack rank).
- Use mapping, forecasting, and refinement to keep it healthy.
- Backlog and board show the same items in different views.
