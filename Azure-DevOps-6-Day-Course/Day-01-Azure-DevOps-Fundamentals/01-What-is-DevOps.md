# 01 - What is DevOps?

## Overview

**DevOps** is a combination of cultural philosophies, practices, and tools that increases an organization's ability to deliver applications and services at high velocity. The name is a blend of **Dev**elopment and **Op**erations — two teams that historically worked in isolation.

> DevOps is **not** a tool, a job title, or a single team. It is a *way of working* that happens to be supported by tools.

## The Problem DevOps Solves

In traditional organizations:

- **Developers** want to ship new features quickly.
- **Operations** want to keep systems stable and avoid change.

This creates a "wall of confusion": developers throw code over the wall, operations struggle to run it, and when something breaks each side blames the other. The result is slow releases, frequent outages, and frustrated teams.

DevOps tears down that wall by making both teams share responsibility for the entire lifecycle — from idea to production and beyond.

## The Three Ways (from *The Phoenix Project* / *The DevOps Handbook*)

1. **Flow** — Optimize the flow of work from development to operations to the customer. Make work visible, reduce batch sizes, and remove bottlenecks.
2. **Feedback** — Create fast feedback loops so problems are detected and fixed early.
3. **Continuous Learning & Experimentation** — Foster a culture of experimentation, learning from failure, and continuous improvement.

## Core Pillars of DevOps (CALMS)

| Letter | Pillar | Meaning |
| ------ | ------ | ------- |
| **C** | Culture | Shared ownership, collaboration, no blame |
| **A** | Automation | Automate builds, tests, deployments, infrastructure |
| **L** | Lean | Eliminate waste, deliver small increments |
| **M** | Measurement | Measure everything — performance, quality, lead time |
| **S** | Sharing | Share knowledge, tools, and responsibility |

## Key Benefits

- **Speed** — release features and fixes faster.
- **Reliability** — automated testing and monitoring catch issues early.
- **Scale** — manage complex systems with infrastructure as code.
- **Collaboration** — developers and operations work as one team.
- **Faster recovery** — smaller changes are easier to roll back.

## Common DevOps Practices

- Continuous Integration (CI) and Continuous Delivery (CD)
- Infrastructure as Code (IaC)
- Microservices architecture
- Monitoring and logging
- Version control for everything
- Automated testing

## Key DevOps Metrics (DORA)

The DORA (DevOps Research & Assessment) metrics measure team performance:

1. **Deployment Frequency** — how often you deploy to production.
2. **Lead Time for Changes** — time from commit to production.
3. **Change Failure Rate** — percentage of deployments causing failures.
4. **Mean Time to Restore (MTTR)** — how quickly you recover from incidents.

## Summary

- DevOps unites Development and Operations through culture, automation, and feedback.
- It exists to deliver software faster *and* more reliably.
- Remember **CALMS** (Culture, Automation, Lean, Measurement, Sharing) and the **Three Ways**.
