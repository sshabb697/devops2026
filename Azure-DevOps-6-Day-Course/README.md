# Azure DevOps 6-Day Course

A complete, hands-on, beginner-to-job-ready course that takes you through the entire Microsoft **Azure DevOps** platform in six focused days. Whether you are an aspiring DevOps engineer, a developer who wants to understand CI/CD, or an administrator responsible for an Azure DevOps organization, this course gives you the concepts, the commands, and the real-world labs you need.

---

## What You Will Learn

By the end of this course you will be able to:

- Explain DevOps culture, principles, and the DevOps lifecycle.
- Navigate and administer an Azure DevOps organization and projects.
- Use **Azure Repos** with Git for version control, branching strategies, pull requests, and branch policies.
- Build and release software with **Azure Pipelines** using both Classic and YAML pipelines.
- Plan and track work with **Azure Boards** (Epics, Features, User Stories, Tasks, Bugs, Sprints, Kanban).
- Configure secure, multi-stage deployments with environments, approvals, checks, and service connections.
- Deliver a complete real-time project from organization creation to production deployment.
- Build, publish, and deploy a **.NET** web application from local machine to **Azure App Service** (see [.NET Deployment Workshop](./DotNet-Deployment-Workshop/)).

---

## Who This Course Is For

- Beginners who are new to DevOps and want a structured path.
- Developers who want to add CI/CD and automation skills.
- System administrators and SREs moving into platform engineering.
- Anyone preparing for the **AZ-400 (Designing and Implementing Microsoft DevOps Solutions)** certification or DevOps interviews.

## Prerequisites

- Basic familiarity with the command line.
- A computer running Windows, macOS, or Linux.
- A free [Azure DevOps](https://dev.azure.com) account (instructions provided in Day 1).
- Optional but helpful: basic programming knowledge in any language.

## How to Use This Repository

1. Start with **Day 1** and work through the numbered lessons in order (`01-`, `02-`, ...).
2. Read the concept lessons, then complete the **Hands-On-Lab** at the end of each day.
3. Use the [Resources](./Resources/) folder for cheat sheets, interview questions, and links.
4. Practice with the [Sample-Applications](./Sample-Applications/) when you reach the pipelines days.

> Tip: Each day's `README.md` is a table of contents and learning-objectives summary for that day.
