# Day 2 — Azure Repos & Git

Today you'll master version control. We start with Git fundamentals (architecture, installation, the essential commands), then move into collaboration with branches, branching strategies, pull requests, and the policies that keep a shared codebase healthy in **Azure Repos**.

## Learning Objectives

By the end of today you will be able to:

- Explain version control and the difference between centralized and distributed systems.
- Describe Git's architecture (working directory, staging area, repository).
- Install and configure Git.
- Use the core Git commands confidently.
- Create and manage branches and apply a branching strategy.
- Open, review, and complete **pull requests**.
- Protect branches with **branch policies**.
- Resolve **merge conflicts**.
- Use **tags**, `.gitignore`, and manage **repository permissions**.
